## 第1部: まずこれだけ — 5〜10分で使い始める

この部分を終えると、TwitchのコメントがOBS上のパネルに表示されるようになります。

---

### ステップ1: ダウンロードページを開く

ブラウザで次のページを開きます。

**https://github.com/ugdoctor/obs-live-hub/releases**
<img width="1320" height="757" alt="image" src="https://github.com/user-attachments/assets/7037f787-824e-434d-9b95-1a5e97bcc684" />

一番上にある項目が最新バージョンです。

---

### ステップ2: ファイルをダウンロードする

最新バージョンの「Assets」（または「ファイル」）の欄をクリックして展開し、
下記のファイルをダウンロードします。

| ダウンロードするファイル | 説明 |
|---|---|
| `obs-live-hub.dll` | プラグイン本体 |
| `obs-live-hub-data.zip`（またはdataフォルダ） | HTMLファイルなどの付属データ |

[ここにスクリーンショット: AssetsのDLLとZIPファイルが見えている画面]

> **ファイルが見当たらない場合:** 「Show all assets」のようなリンクをクリックすると
> 全ファイルが表示されます。

ダウンロードしたZIPファイルは、デスクトップなど分かりやすい場所に解凍（展開）してください。

---

### ステップ3: プラグインをOBSに追加する

> **注意:** この手順では管理者権限が必要なフォルダにファイルをコピーします。
> 「続行しますか？」のダイアログが出たら「はい」を選んでください。

**3-1. プラグイン本体（DLL）のコピー**

`obs-live-hub.dll` を次のフォルダにコピーします。

```
C:\Program Files\obs-studio\obs-plugins\64bit\
```

コピー手順：
1. エクスプローラーでダウンロードした `obs-live-hub.dll` を右クリック → 「コピー」
2. アドレスバーに `C:\Program Files\obs-studio\obs-plugins\64bit\` と入力してEnter
3. 空いている場所を右クリック → 「貼り付け」

[ここにスクリーンショット: obs-plugins\64bitフォルダにDLLをコピーした状態]

> **OBSをSteam版でインストールした場合など、フォルダが違う場合:**  
> OBSのインストール先フォルダ（例: `D:\Steam\steamapps\common\obs-studio\`）の中の
> `obs-plugins\64bit\` にコピーしてください。

**3-2. データファイルのコピー**

解凍したフォルダの中にある HTMLファイル（`overlay.html` / `tts.html` / `effect.html` / `debug.html`）を、
DLLの隣にある `obs-live-hub` という名前のサブフォルダに入れます。

```
C:\Program Files\obs-studio\obs-plugins\64bit\obs-live-hub\
```

コピー手順：
1. エクスプローラーで `C:\Program Files\obs-studio\obs-plugins\64bit\` を開く
2. `obs-live-hub` という名前のフォルダを新規作成する（DLL と同じ階層）
3. 解凍したファイルの中の HTMLファイル（`overlay.html` / `tts.html` / `effect.html` / `debug.html`）を、
   作成した `obs-live-hub` フォルダの中にコピーする

[ここにスクリーンショット: obs-live-hubサブフォルダの中にHTMLファイルが入っている状態]

完了後、フォルダの中身はこのようになっているはずです。

```
C:\Program Files\obs-studio\obs-plugins\64bit\
    obs-live-hub.dll
    obs-live-hub\
        overlay.html
        tts.html
        effect.html
        debug.html
```

> **自動コピーについて:** OBS起動時にプラグインが自動的にこれらのファイルを
> `%APPDATA%\obs-studio\plugins\obs-live-hub\` にコピーします。
> 手動で APPDATA フォルダにコピーする必要はありません。

### ステップ5: Twitchのコメントを表示する

Twitchのコメントを表示するには、一度だけ「Twitchアプリの登録」という作業が必要です。
5分ほどで終わりますので、順番に進めてください。

**5-1. Twitch Developer Consoleでアプリを登録する**

1. ブラウザで **https://dev.twitch.tv/console** を開く
2. Twitchアカウントでログインする（普段使っているアカウントでOK）
3. 「アプリケーション」タブをクリック → 「アプリケーションを登録する」ボタンをクリック

[ここにスクリーンショット: Twitch Developer Consoleのアプリ登録画面]

4. 以下のように入力する：
   - **名前（Name）**: 何でもOK（例: `obs-live-hub`）
   - **OAuth リダイレクト URL**: `http://localhost` と入力して「追加」ボタンをクリック
   - **カテゴリ（Category）**: 「その他」や「ゲーム統合」など何でもOK

5. 「作成」ボタンをクリック

6. 作成したアプリの「管理」ボタンをクリック

[ここにスクリーンショット: アプリ登録後の管理画面]

7. 画面に表示されている **クライアントID（Client ID）** をメモしておく  
   （「新しいシークレット」は今回は不要です）

**5-2. OBSで設定する**

1. OBSの「ツール」→「obs-live-hub」→「接続・設定」→「設定」をクリック

[ここにスクリーンショット: 設定ダイアログが開いた画面]

2. **Twitch Channel** の欄に、自分のTwitchのチャンネル名（配信URL `https://twitch.tv/○○` の ○○ 部分）を入力

3. **Twitch Client ID** の欄に、先ほどメモしたクライアントIDを貼り付ける

4. **「Twitchアカウントと連携」** ボタンをクリック

5. ブラウザが開いてTwitchのログイン画面が表示される。ログインして「許可」ボタンをクリック

6. OBSに戻り、「連携済み」と表示されていることを確認する

7. 「OK」ボタンをクリックして設定ダイアログを閉じる

> 「OK」を押すと自動的にTwitchへの接続を試みます。接続できない場合は
> 「接続・設定」→「リロード」を選択してください。

**5-3. 動作確認する**

1. Twitchでコメントを送ってみる（自分のチャンネルで何かコメントしてみてください）

2. OBSのドックパネル（「**Comment Viewer**」）にコメントが表示されれば成功です

[ここにスクリーンショット: OBSのドックパネルにコメントが表示されている画面]

> **ドックパネルが見当たらない場合:**  
> OBSのメニューから「表示」→「ドック」→「Comment Viewer」にチェックを入れてください。

### コメントをOBS画面に重ねて表示する（オーバーレイ）

OBSのブラウザソース機能を使うと、コメントを配信画面に直接重ねて表示できます。

1. OBSのソースリストで「＋」をクリック → 「ブラウザ」を選択

[ここにスクリーンショット: ブラウザソースの追加画面]

2. 「ローカルファイル」のチェックを入れる

3. 「参照」ボタンをクリックして、次のファイルを選択する

   ```
   %APPDATA%\obs-studio\plugins\obs-live-hub\overlay.html
   ```

   > 上のパスをそのままコピーしてファイル選択のアドレスバーに貼り付けると便利です

4. 幅・高さはシーンに合わせて設定する（例: 400×900）

5. 「OK」をクリック

これで、コメントがOBSの画面に重なって表示されるようになります。

---

### コメントを読み上げる（TTS）

「TTS（Text-To-Speech）」は、コメントをコンピューターの声で読み上げる機能です。

#### まず試してみる — Web Speech API（追加インストール不要）

Web Speech API はブラウザに内蔵された音声機能です。追加のソフトウェアが不要で、
すぐに使い始めることができます（音質はシンプルです）。

1. OBSのソースリストで「＋」→「ブラウザ」を追加

2. 「ローカルファイル」にチェックを入れ、次のファイルを選択する

   ```
   %APPDATA%\obs-studio\plugins\obs-live-hub\tts.html
   ```

3. このブラウザソースの「音声モニタリング」を「モニターと出力」または「モニターのみ」に設定する  
   （ソースを右クリック → 「音声フィルタ」→「詳細音声設定」から設定）

4. OBSの「ツール」→「obs-live-hub」→「読み上げ」→「読み上げ設定」を開く

5. 「Web Speech API」の項目で「有効」にチェックを入れて、「OK」をクリック

6. Twitchにコメントを送ってみて、音声が再生されれば成功です

[ここにスクリーンショット: 読み上げ設定ダイアログの画面]

#### 棒読みちゃんを使う（無料・高品質）

棒読みちゃんは無料で使えるWindowsの音声読み上げソフトです。

**インストール手順:**

1. 「棒読みちゃん」で検索し、公式サイトからダウンロードしてインストールする

2. 棒読みちゃんを起動しておく（配信中は起動しておく必要があります）

3. OBSの「ツール」→「obs-live-hub」→「読み上げ」→「読み上げ設定」を開く

4. 「棒読みちゃん」の項目で「有効」にチェックを入れる

5. 「実行ファイルパス」に棒読みちゃんのインストール先（`BouyomiChan.exe`）を設定し、
   「OBS起動時に自動起動する」にもチェックを入れると便利

6. 「OK」をクリック

> **音が聞こえない場合:** [第3部「読み上げが動かない・聞こえない」](#読み上げが動かない聞こえない) を参照してください。

#### 視聴者が自分の声を変えるには

視聴者はチャットに `[olh] engine:bouyomi` のようにコメントを送ることで、
自分のコメントの読み上げ声を変えることができます。

詳しい一覧は [視聴者向けコマンド](CLAUDE.md) を参照してください（開発者向け文書ですが、コマンド一覧として参照できます）。

### YouTubeのコメントも表示する

YouTube連携にはGoogleのアカウント設定が必要です。初回のみ設定が必要で、
慣れれば20〜30分で完了します。

> **⚠️ 難易度: 中程度**  
> Google Cloud Console という開発者向けのサービスを使います。
> 画面の文字が英語で表示される場合がありますが、ガイドの手順通り進めれば完了できます。

**手順1: Googleのプロジェクトを作成する**

1. ブラウザで **https://console.cloud.google.com/** を開く
2. Googleアカウントでログインする（YouTubeと同じアカウント）
3. 画面上部の「プロジェクトを選択」をクリック → 「新しいプロジェクト」をクリック
4. プロジェクト名を入力（例: `obs-live-hub`）→ 「作成」をクリック

[ここにスクリーンショット: Google Cloud Consoleのプロジェクト作成画面]

**手順2: YouTube APIを有効にする**

1. 左メニューから「APIとサービス」→「ライブラリ」をクリック
2. 検索欄に「YouTube Data API v3」と入力して検索
3. 検索結果をクリック → 「有効にする」ボタンをクリック

**手順3: OAuth認証情報を作成する**

1. 左メニューから「APIとサービス」→「認証情報」をクリック
2. 「認証情報を作成」→「OAuth クライアント ID」をクリック
3. 「同意画面を構成」というボタンが出た場合はクリックして、以下のように設定する：
   - ユーザーの種類: 「外部」を選択 → 「作成」
   - アプリ名: 何でもOK（例: `obs-live-hub`）
   - サポートメール: 自分のGmailアドレス
   - その他はデフォルトのまま → 「保存して次へ」を何度かクリック → 「ダッシュボードに戻る」
4. 「認証情報を作成」→「OAuth クライアント ID」をクリック
5. アプリケーションの種類: 「デスクトップアプリ」を選択
6. 名前: 何でもOK → 「作成」をクリック
7. **クライアントID** と **クライアントシークレット** をメモしておく

[ここにスクリーンショット: OAuth認証情報の作成完了画面]

**手順4: OBSで設定する**

1. OBSの「ツール」→「obs-live-hub」→「接続・設定」→「設定」を開く

2. 「YouTube API Key」欄に、手順2で取得したAPIキーを入力する

   > **APIキーの取得方法:** Google Cloud Consoleの「APIとサービス」→「認証情報」→「認証情報を作成」→「APIキー」で作成できます

3. 「Google OAuth Client ID」に、手順3でメモしたクライアントIDを入力

4. 「Google OAuth Client Secret」に、クライアントシークレットを入力

5. 「Googleアカウントと連携」ボタンをクリック → ブラウザが開くのでGoogleアカウントでログイン → 「許可」をクリック

6. OBSに戻り「連携済み」と表示されていれば成功

7. 「YouTube Broadcast ID」の欄は `me`（デフォルト）のままでOK。配信開始時に自動的に配信中の動画が検索されます

> **⚠️ 「テスト中」ステータスの7日間制限について**
>
> Google Cloudのアプリを作ったばかりの状態（公開していない状態）では、
> **7日間OBSを起動しないと認証が切れてしまいます。**
>
> 認証が切れた場合は「接続・設定」→「設定」を開き、「Googleアカウントと連携」ボタンを押してもう一度ログインしてください。
> 再ログインは1〜2分で完了します。

### X（旧Twitter）へ投稿する

#### X手動投稿（かんたん・推奨）

X（旧Twitter）のアカウントがあれば、APIキーの登録なしで投稿できます。
ブラウザで投稿画面を開いて、自分で「投稿」ボタンを押す方式です。

1. OBSの「ツール」→「obs-live-hub」→「X投稿」→「X手動投稿」をクリック

[ここにスクリーンショット: X手動投稿ダイアログが開いた画面]

2. 投稿したい本文を入力する

3. 配信URLを追加したい場合は「Twitchリンク」「YouTubeリンク」のチェックをONにする
   （あらかじめ各プラットフォームの設定が完了している場合のみ有効です）

4. 「ブラウザで投稿」ボタンをクリック

5. ブラウザで `x.com/intent/post` が開くので、内容を確認して「投稿」をクリック

#### テンプレートを設定しておくと便利

毎回同じような文章を投稿する場合は、あらかじめテンプレートを登録しておくと
ワンクリックで本文が入力された状態でダイアログが開きます。

1. 「ツール」→「obs-live-hub」→「X投稿」→「Xテンプレート設定」をクリック
2. 「追加」ボタンでテンプレートを作成し、「名前」と「投稿本文」を入力
3. 「保存」をクリック

#### 配信開始時に自動でダイアログを表示する

「配信開始ボタンを押したら自動でX投稿画面を開く」ように設定できます。

1. 「ツール」→「obs-live-hub」→「X投稿」→「配信開始時の自動投稿設定」をクリック
2. 「手動投稿ダイアログを表示」を選択して「OK」をクリック

これで、次回から配信を開始するとX手動投稿のダイアログが自動的に開きます。

### ポイント機能と視聴者コマンドの基本

視聴者がチャットに特定のコマンドを送ることで、いろいろな操作ができます。

#### 視聴者ができる操作（例）

| チャットに送るコマンド | 効果 |
|---|---|
| `[olh] color:red` | 自分のコメント文字を赤色に変える |
| `[olh] engine:bouyomi` | 自分のコメントを棒読みちゃんで読み上げてもらう |
| `[olh] point_check` | 自分の現在のポイント残高を確認する |
| `!vote A` | 開催中のアンケートに「A」で投票する |

#### ポイントシステムを設定する

1. 「ツール」→「obs-live-hub」→「ポイント」→「ポイント設定」をクリック
2. 「ポイントシステムを有効にする」にチェックを入れる
3. コメントごとに付与するポイント数や、ポイントで使えるアクションを設定する

## 第3部: 困ったときは

---

### 「ツールメニューにobs-live-hubが出てこない」

**確認すること1: DLLが正しいフォルダに入っているか**

エクスプローラーで次のフォルダを開き、`obs-live-hub.dll` があるか確認してください。

```
C:\Program Files\obs-studio\obs-plugins\64bit\
```

**確認すること2: OBSのバージョンが古くないか**

OBS Studio 31.x 以降が必要です。OBSを最新バージョンにアップデートしてください。

**確認すること3: OBSを再起動したか**

DLLを追加した後にOBSを起動している場合は、再起動してください（プラグインはOBS起動時にのみ読み込まれます）。

**確認すること4: OBSのログを確認する**

OBSの「ヘルプ」→「ログファイル」→「現在のログファイルを表示」を開き、
`obs-live-hub` という文字が含まれている行を探します。

- `loaded module 'obs-live-hub.dll'` のような行があれば読み込み成功
- `Failed to load` などのエラーが出ていれば、DLLのコピー先が間違っているか、依存関係の問題がある可能性があります

---

### 「読み上げが動かない・聞こえない」

**手順1: tts.html のブラウザソースが追加されているか確認する**

OBSのソースリストに `tts.html` を使ったブラウザソースがあるか確認してください。
ない場合は [「コメントを読み上げる」](#コメントを読み上げるtts) の手順を参照してください。

**手順2: 音声モニタリングが設定されているか確認する**

tts.html のブラウザソースの音声モニタリングが「モニターと出力」または「モニターのみ」になっているか確認してください。
（ソースを右クリック → 「詳細音声設定」）

**棒読みちゃんが使えない場合**

1. 棒読みちゃんが起動しているか確認する（タスクバーに表示されているはず）
2. 「ツール」→「obs-live-hub」→「読み上げ」→「読み上げ設定」を開き、棒読みちゃんが「有効」になっているか確認
3. OBSのログ（「ヘルプ」→「ログファイル」）に `BouyomiChanClient` や `speak OK` などの文字があるか確認する

---

### 「YouTubeでエラーが出る」

**「invalid_grant」というエラーが出る場合**

これは認証情報が切れているサインです。「接続・設定」→「設定」を開き、「Googleアカウントと連携」からもう一度ログインしてください。

このエラーは「テスト中」ステータスのアプリで7日間以上OBSを起動しなかった場合に発生します。
再ログインすれば解決します。

**「HTTP 401」や「quota exceeded」というエラーが出る場合**

- **401**: 認証が切れています。上記と同様に再ログインしてください
- **quota exceeded（クォータ超過）**: YouTube APIの無料利用上限（1日あたり）に達しています。
  翌日になれば自動的にリセットされます。Google Cloudで課金を有効にしている場合は、
  「接続・設定」→「設定」の「**課金を有効化済み・クォータ上限を無視する**」にチェックを入れると制限なく継続できます

**YouTubeコメントが表示されない（エラーなし）**

1. YouTube Studioで配信が「ライブ中」になっているか確認
2. 「接続・設定」→「リロード」をクリックして再接続を試みる
3. コメントが実際に投稿されているか確認（非常に低頻度のポーリングのため、表示まで数秒かかります）

### よくある質問

**Q. Twitchのアカウントで配信していないのにコメントが見えますか？**

A. はい。配信中でなくても、チャンネルが存在してチャットが有効であれば表示されます。
   ただし、通常はコメントを受け取るには配信中（またはフォロワー専用チャット解除中）である必要があります。

**Q. コメントがリアルタイムで表示されますか？**

A. Twitchはほぼリアルタイム（1〜2秒以内）です。
   YouTubeはAPIのポーリング（定期確認）方式のため、コメントが表示されるまで数秒〜10秒程度かかります。

**Q. 複数の配信プラットフォームを同時に使えますか？**

A. はい。TwitchとYouTubeの両方を同時に接続できます。コメントは一つのリストにまとめて表示されます。

**Q. 視聴者が `[olh]` コマンドを送っても反応しない**

A. まず「接続・設定」→「設定」でその配信プラットフォームに接続されているか確認してください。
   次に、そのコマンドが有効なエンジン・機能が「読み上げ」→「読み上げ設定」や「ポイント」→「ポイント設定」で有効になっているか確認してください。

**Q. プラグインをアンインストールするには？**

A. 次の2ファイルを削除してください。
   - `C:\Program Files\obs-studio\obs-plugins\64bit\obs-live-hub.dll`
   - `%APPDATA%\obs-studio\plugins\obs-live-hub\`（フォルダごと削除）

   設定ファイルは `%APPDATA%\obs-studio\plugin_config\obs-live-hub\config.json` に保存されています。
   設定も消したい場合はこのファイルも削除してください。

**Q. Xへの投稿でX APIキーは必要ですか？**

A. 「X手動投稿」機能（Web Intent方式）はAPIキー不要です。
   ブラウザを経由して投稿するため、X（旧Twitter）のアカウントがあれば誰でも使えます。
